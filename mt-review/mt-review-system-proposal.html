[ 01 ] OVERVIEW

왜 음성 우선인가

기존 MT 복습과의 본질적 차이는 학습 메커니즘 자체에 있다

POLY 학생들은 매달 MT(Monthly Test)를 본다. 시험이 끝나면 틀린 문제를 다시 푼다. 선생님이 설명하고, 학생은 받아 적고, 다음 시험까지 잊어버린다. 이 흐름은 수십 년 동안 바뀌지 않았다.

MT Review는 이 흐름을 끊는다. 학생은 Luna와 음성으로 대화하면서 자신이 왜 그 답을 골랐는지 직접 설명한다. 정답을 맞춰도 Luna는 한 번 더 묻는다 — "어떤 부분을 읽고 그렇게 생각했어?"

지문은 정형화된 문제 은행에서 꺼내오지 않는다. 학생의 오답 데이터와 흥미·레벨을 바탕으로 AI가 매번 새로 가공해 제시한다 — 같은 Skill을 다른 이야기로 반복 노출시켜 "또 이 문제네" 하는 학습 권태를 차단한다.

차원 기존 MT 복습 MT Review
학습 주체 선생님이 설명, 학생은 수동적으로 수용 Luna가 질문, 학생이 스스로 언어로 설명
피드백 구조 문제 풀기 → 채점 → 끝 대화 → 이해 확인 → 칭찬 → 다음 문제
오답 처리 정답·오답 이분법 "왜 그렇게 생각했어?" 단계적 이끌기
개인화 모든 학생 동일 경험 학생 오답 데이터 기반 맞춤 지문 생성
지문 소스 고정된 문제 은행에서 추출 학생 흥미·레벨 반영 AI 실시간 생성
한 줄 정리

MT Review는 "문제지 다시 풀기"가 아니라 "왜 그렇게 답했는지 말로 설명하기"다. 영어를 쓰는 게 아니라 영어로 생각하게 만드는 학습이다.

[ 02 ] UX PHILOSOPHY

3가지 핵심 철학

화면을 비우고, 정답을 비우고, 숙제라는 느낌을 비운다
01 · Voice First

화면이 아닌
음성으로 배운다

입력창도, 채팅 버블도, 캐릭터 이미지도 없다. 학생은 손가락이 아닌 입으로 답하고, Luna도 텍스트 말풍선 없이 목소리로만 존재한다.

"화면을 비워 배경 영상과 지문에 집중하게 만든다."
02 · No "Wrong"

틀렸다는 말은
절대 하지 않는다

Luna는 "틀렸어", "오답", "No"라는 표현을 쓰지 않는다. 대신 지문으로 돌아가는 질문만 한다. 정답을 맞춰도 한 번 더 "왜?"를 묻는다.

"근거를 언어로 표현하는 훈련을 시킨다."
03 · Game Feel

복습이 게임
한 판처럼

지문 키워드 기반 배경 영상이 자동 전환되고, 정답 시 컨페티가 터지며, 진행바가 골드 스타로 채워진다. 숙제가 아닌 짧은 게임이 된다.

"0점은 없다 — 힌트를 써도 1점은 받는다."

[ 04 ] VOICE EXPERIENCE

음성은 핵심 인터페이스다

화면 UI보다 더 많은 설계 고민이 담겨 있는 영역

Luna는 한국어와 영어를 자연스럽게 전환한다. 영어 음성은 pitch 0.92로 낮춰 어린 목소리가 아닌 신뢰감 있는 선생님 느낌을 구현했다. 기본 TTS 음성이 너무 앳된 느낌을 주는 문제를 해결한 의도적 튜닝이다.

격려 · 설명 · 질문

KO

따뜻하고 친근한 톤으로, 학생의 첫 시도를 인정하고 다음 단계로 자연스럽게 이끈다. 칭찬은 구체적으로, 힌트는 부드럽게.

언어 한국어 rate 0.92 톤 친근 · 따뜻

지문 · 문제 낭독

EN

UK Female 음성, pitch를 낮춰 차분한 선생님 톤으로 조정. 학생이 글자를 읽는 동시에 정확한 영어 발음을 흘려듣게 한다.

언어 English (UK) pitch 0.92 톤 차분 · 신뢰

STT(음성 인식) 설계 결정

브라우저 기반 STT는 특히 모바일에서 제약이 많다. 다음은 실제 학원 환경에서 검증된 주요 설계 결정 사항이다.

실전 STT 가드레일 4가지

1. 탭해서 말하기 버튼 — 모바일에서 사용자 제스처 없이 마이크를 시작하면 브라우저가 차단한다. 자동 시작 대신 탭 버튼으로 명시적 트리거.

2. 디바이스별 침묵 타이머 — 모바일 10초 / 데스크탑 5초. 학생이 생각하고 영어 문장을 떠올리는 시간을 충분히 확보.

3. interim fallback — Android Chrome은 최종 결과 없이 끝나는 경우가 있다. 임시 결과를 메모리에 저장해뒀다가 최종 누락 시 사용.

4. 아동 음성 특화 평가 — 1~3학년은 발음·억양이 부정확하다. 성인 기준의 strict 매칭 대신 음소 유사도 + LLM 의미 평가를 병행 — 문법은 틀려도 의도가 맞으면 정답으로 처리한다.

기술 아키텍처

학습 효과를 만드는 건 UI가 아니라 그 뒤의 기술 스택이다. 세 가지 핵심 기술이 "할루시네이션 없는 안전한 지문 생성", "교감 가능한 음성", "아동 발음에 관대한 평가"를 가능하게 한다.

01 · RAG

POLY 커리큘럼 결합 검색증강생성

초등 저학년에게 부적절하거나 검증되지 않은 문장이 생성되는 할루시네이션을 원천 차단한다. POLY 고유 커리큘럼 데이터와 LLM을 결합해 Skill 기반 안전한 지문을 실시간 생성한다.

02 · Emotional TTS

교감 가능한 Luna 음성

딱딱한 기계음이 아닌 정서적 표현이 가능한 TTS 엔진으로 모델링. 칭찬·격려·질문 각각의 톤이 다르게 들리도록 SSML 태그로 미세 조정 — Phase A에 ElevenLabs 도입 예정.

03 · Child STT

아동 음성 특화 평가

발음·억양이 부정확한 1~3학년에 맞춰 strict 매칭 대신 의미 평가 적용. 음소 유사도 + LLM 의도 판별을 병행해 "rain fell hard"를 "lane fell hard"로 인식해도 정답 처리.

◆ Expected Impact

"시험"이 아닌 상호작용형 콘텐츠로의 패러다임 전환

MT Review는 1~3학년 학생에게서 "평가"라는 심리적 부담을 걷어낸다. 친근한 Luna와 한 편의 짧은 이야기를 완성하는 성취감을 제공함으로써, 오답 관리가 자기주도적인 즐거운 여정으로 변모한다.

학부모는 "우리 아이가 영어로 말한다"는 만족을, 선생님은 "수업 외 시간에도 학생이 학습한다"는 운영 효율을, 학원은 다음 MT 성적 향상이라는 실질적 성과를 얻는다.
